2. Executive Summary
Andromeda is a next-generation product discovery and comparison platform that aggregates products from multiple marketplaces, local businesses, and independent sellers into a single unified ecosystem. Inspired by the vast Andromeda Galaxy, the platform aims to become the universe of product discovery — where users search once and discover everything.

The platform directly addresses fragmented online shopping by providing real-time price comparison, side-by-side product evaluation, local business visibility, and AI-powered recommendations through a single, intuitive interface.

This PRD defines the full technical requirements, feature specifications, user stories, and architectural decisions for the Andromeda platform built on the Next.js 15 full-stack framework with TypeScript, Tailwind CSS, and a FastAPI microservice backend.

6.2 Next.js 15 Architecture Decisions
App Router (Mandatory)
	•	Use the App Router exclusively — Pages Router is prohibited
	•	All routes defined in the app/ directory using folder-based routing conventions
	•	Route groups used for layout segregation: (marketing), (dashboard), (auth), (seller)
	•	Parallel routes for modal-style product comparison overlay on listing pages

Server vs Client Components
	•	Default to React Server Components (RSC) — fetch data server-side where possible
	•	'use client' directive used only for interactive components (search bar, comparison widget, cart, alerts)
	•	Co-locate client components with their RSC parents to minimise JavaScript bundle
	•	Streaming with Suspense boundaries for product grids and recommendation feeds

Data Fetching Strategy
	•	Server Components: native fetch() with Next.js cache tags for product listings and categories
	•	Static pages (ISR): category and seller pages revalidated every 60 seconds
	•	Dynamic pages: product detail and comparison pages streamed via Suspense
	•	Client-side: TanStack Query for search autocomplete, wishlist, cart, and real-time stock
	•	Server Actions for all mutations: review submit, seller onboarding, alert creation

12. Security Requirements
	•	All protected API routes validated via NextAuth.js session or verified JWT in middleware
	•	Role-based access control (RBAC): user / seller / admin roles enforced at middleware and route level
	•	Input validation via Zod schemas on all Server Actions and Route Handlers
	•	SQL injection protection via parameterised queries through Prisma or SQLAlchemy
	•	XSS prevention: React default escaping plus Content Security Policy (CSP) headers
	•	CORS configured to allow requests only from Andromeda-owned domains
	•	Seller product listings require admin or automated review before going live
	•	Fake review detection using NLP heuristics, velocity checks, and anomaly detection
	•	GDPR-compliant: user data export and account deletion endpoints provided
	•	All secrets managed via Vercel Environment Variables or Doppler — never committed to source
	•	Dependency scanning via GitHub Dependabot; security patches applied within 48 hours

	•	Build verification: next build must complete without errors

16. Assumptions, Constraints & Risks
16.1 Assumptions
	•	Third-party marketplace APIs (Amazon, Flipkart) will remain stable and accessible
	•	Target users have access to modern browsers supporting ES2020+ features
	•	Sellers have basic digital literacy sufficient to navigate the onboarding flow
	•	Meilisearch cloud tier will support MVP search volume without degradation

16.2 Constraints
	•	MVP must be delivered within 3 months with a development team of 4 engineers
	•	Initial budget limits AI recommendation features to Phase 3 and beyond
	•	Platform must comply with Indian IT Act, IT Rules 2021, and consumer protection regulations
	•	No native mobile application in scope for v1.0 — responsive web only
